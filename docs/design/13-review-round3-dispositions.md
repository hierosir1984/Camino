## Deferrables (10) — most addressed in v4 text, remainder registered for the PRD

- **8 Freshness liveness/races** — CONFIRMED. v4: serialize-per-repo restored explicitly (round 3 caught its silent loss), making the merge queue trivially depth-one; bounded revalidation retries then escalate; mission-branch protection + non-bypass settings listed as onboarding requirements; "scoped revalidation" defined as re-execution (old verdicts expire, never rebind). Residual detail (exact retry bounds) → PRD registry.
- **9 Path normalization/.gitattributes** — CONFIRMED. Folded into quarantine v2 (blocker 2).
- **10 Canon-neutral classification still gates folds via one creative judgment** — CONFIRMED. v4: quick tasks always get a cross-family mini-review (cheap at that scale); deterministic user-observable heuristics (route/UI/API-surface paths); fold suppression requires planner *and* reviewer agreement. Residual heuristic list → PRD registry.
- **11 The 50-agreement bound assumes stationary trials** — CONFIRMED. v4 wording: the bound is stated as applying to past sampled work only; unlocks framed as policy with rolling windows, heterogeneity acknowledged, revocation instant. No reliability claim about future distribution-shifted work.
- **12 Mutation score ≠ probe false-negative estimator** — CONFIRMED (literature agrees). v4: mutation testing repositioned as probe-improvement tooling; tier-2 prerequisite becomes accumulated post-merge outcome tracking instead.
- **13 knowledge.md lacks a trust lifecycle** — CONFIRMED. v4: candidate/approved entry states; attempts write candidates with provenance; fold review promotes; expiry and contradiction handling named; packs include approved entries only (invariant 2 preserved).
- **14 No mission/issue/attempt state machine, leases, or serialization** — CONFIRMED (and 00 required it). v4: state machine sketch with attempt leases, resume-vs-duplicate rules, single validation-env owner per repo, janitor respects leases; serialize-per-repo restored. Full transition table → PRD.
- **15 Router treats branch merge as terminal** — CONFIRMED. v4: trajectories terminal only at mission resolution (main merge / descope / abandonment); branch merges are intermediate; blocked-age now triggers the abandonment penalty at a threshold rather than merely being recorded.
- **16 No path to the claimed autonomy end state; no attention thresholds** — CONFIRMED. v4: plan-approval and quick-task tiers added to the autonomy ladder as later stages with the same earned mechanics; attention budget gets explicit provisional numbers and a defined overrun trigger. Exact numbers → PRD registry.
- **17 Evidence packet reduced to scattered references** — CONFIRMED (00 required it; two rounds flagged it). v4: evidence packet restored as a first-class object (contract version, requirement map, commands, artifacts, retries, verdicts; bound to attempt + SHAs); v0 GUI scope gains an evidence viewer.
